# 硕士申请资料分析工作流程

## 目标
帮助用户整理硕士项目课程的选课要求、课程列表和课程描述，用于硕士申请准备。

## 工作区
`d:/本科/MS` — 每个学校/专业整理成一个独立的 `.md` 文件。

## 用户感兴趣的内容（仅提取以下内容，忽略其他）
1. **修读/选课要求** — 学分要求、必修/选修分布、毕业条件等
2. **课程代码和名称** — 每门课的具体代码（如 CS 601）和全称
3. **课程描述** — 如有提供，提取课程简介

## 支持的源类型

### 类型 A：直接 PDF 链接
- `.pdf` 结尾的链接，Google Drive 分享链接，Box 分享链接
- 用 `curl -x http://127.0.0.1:7897` 下载后用 `pdftotext` 分页提取

### 类型 B：HTML 网页
- 课程描述网页、项目要求网页、在线课程目录
- 用 `WebFetch` 工具获取网页内容
- 也可用 `curl -x http://127.0.0.1:7897` 下载 HTML 后用文本工具解析

### 类型 C：链接枢纽文档
- 某些 PDF 或网页内部嵌入了其他课程链接
- **先提取文本 → 识别所有子链接 → 逐一访问子链接**
- Coordinator 负责发现链接，分派给 subagent 逐个访问

## 工作流程

### 通用流程
1. **识别源类型**：PDF / 网页 / 链接枢纽
2. **获取内容**：
   - PDF：`curl -x proxy` 下载 → `pdftotext -f N -l M` 分页提取
   - 网页：`WebFetch` 或 `curl` 获取
3. **浏览前几页/段落**：找到目录或导航结构
4. **定位目标内容**：修读要求、课程列表所在位置
5. **提取识别**：按需获取，避免一次性拉取全部
6. **整理输出**：生成中文说明 + 中英文对照课程表

### 链接枢纽特殊流程
1. 先提取文档全文或关键段落
2. 识别所有疑似课程/要求相关的超链接
3. Coordinator 创建链接清单
4. 并行分派 subagent 访问各链接
5. 汇总所有子页面内容

## 输出文件格式
文件命名：`{学校名}-{专业名}-课程介绍.md`

内容结构：
- 学校/专业名称
- 修读要求（中文解释）
- 课程列表（中英文对照表格：代码、英文名、中文译名、学分、描述）

## 代理设置
- 访问国外网站走本地代理：`127.0.0.1:7897`
- curl 使用 `-x 127.0.0.1:7897` 或 `export https_proxy=http://127.0.0.1:7897`

## 团队架构
```
Coordinator（主线程）
   ├── Link Scout（可选，链接枢纽专用）— 发现并提取子链接
   ├── Requirements Extractor — 提取修读/选课要求，中文解释
   └── Course Extractor — 提取课程代码/名称/描述，中英对照
```
